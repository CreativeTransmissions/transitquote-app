# refresh-app — show TQApp changes on the emulator

There are TWO paths. Pick by what you need. **Default to the fast path** for interactive UI work.

## Mental model (important — avoids a known misdiagnosis)
This is a **debug** build with developer support ON (no `getJSBundleFile()` override in
`MainApplication.kt`, no `expo-updates`, no `expo-dev-client`). So:
- When **Metro is reachable**, the app loads JS **live from Metro** → Fast Refresh on every save,
  no rebuild.
- The embedded `android/app/src/main/assets/index.android.bundle` is only a **fallback**, used
  when Metro is **un**reachable. It does NOT override a reachable Metro.
- On this machine Metro is often unreachable because **Avast Web Shield blocks the
  emulator→Metro path**. That — not the bundle — is what forces stale UI and full rebuilds.
- Do NOT delete the embedded bundle: `android/` is fully gitignored (regenerated by
  `expo prebuild`) so deletion is futile, and the bundle is a hard requirement for Maestro
  (see docs/SMOKE_TESTING.md §2 — `clearState` + Avast-blocked Metro).

## Before any command
- Run from the project dir: `cd C:/Users/andre/AndroidStudioProjects/TQApp` (session cwd is the
  parent `AndroidStudioProjects`).
- `export MSYS_NO_PATHCONV=1` on any line with `adb` + a device path (`/sdcard/...`), or Git-Bash
  rewrites it to `C:/Program Files/Git/sdcard/...`.

## FAST PATH — Metro + Fast Refresh (default for JS/UI/styling work)

Gets edits on screen in seconds, no rebuild.

1. **Avast Web Shield OFF** for the session (it blocks emulator→Metro). Ask the user to toggle it
   if it's on — you can't disable it programmatically. This is the single prerequisite.
2. Start Metro (run_in_background): `npx expo start` (Metro serves on 8081).
3. Wire the emulator to Metro and relaunch:
   ```bash
   cd C:/Users/andre/AndroidStudioProjects/TQApp; export MSYS_NO_PATHCONV=1; \
   adb reverse tcp:8081 tcp:8081 && \
   adb shell am force-stop com.transitteam.app; sleep 1; \
   adb shell monkey -p com.transitteam.app -c android.intent.category.LAUNCHER 1 >/dev/null 2>&1
   ```
4. From now on, **just save the file** — Fast Refresh applies it. If a change doesn't appear,
   trigger a reload: `adb shell input text "RR"` won't work generically; instead re-run the
   force-stop+monkey relaunch above (re-pulls from Metro).
5. Screenshot with the `android-shot` skill.

**One-time confirmation** (do once per machine/session if unsure Fast Refresh is live): edit a
visible string, save, confirm it updates with no rebuild. If it still shows old UI with Metro
reachable + `adb reverse` set + Avast off, something machine-specific is forcing the bundle —
investigate before assuming the fast path works.

## REBUILD PATH — embed bundle + assembleDebug (Maestro / standalone / device)

Use only when the app must run **without** a live Metro: Maestro E2E, sharing an APK, on-device
testing. This is the slow loop (minutes) — run the build in the background.

1. Type-check first: `npx tsc --noEmit 2>&1 | head -20`
2. Regenerate the embedded bundle (entry is `index.ts`):
   ```bash
   cd C:/Users/andre/AndroidStudioProjects/TQApp && \
   NODE_TLS_REJECT_UNAUTHORIZED=0 npx expo export:embed --platform android --dev false \
     --entry-file index.ts \
     --bundle-output android/app/src/main/assets/index.android.bundle \
     --assets-dest android/app/src/main/res 2>&1 | tail -5
   ```
   Verify the new bundle contains your change before building (avoids stale-UI dead-ends):
   `grep -c "<a token from your change>" android/app/src/main/assets/index.android.bundle`
3. Build (run_in_background — minutes):
   ```bash
   cd C:/Users/andre/AndroidStudioProjects/TQApp/android && \
   NODE_TLS_REJECT_UNAUTHORIZED=0 ./gradlew :app:assembleDebug --console=plain
   ```
   If the app must reach DDEV over HTTPS, prefix with the truststore:
   `export JAVA_TOOL_OPTIONS="-Djavax.net.ssl.trustStore=$USERPROFILE/.tqapp-certs/cacerts -Djavax.net.ssl.trustStorePassword=changeit"`
4. Install + relaunch:
   ```bash
   cd C:/Users/andre/AndroidStudioProjects/TQApp; export MSYS_NO_PATHCONV=1; \
   adb install -r android/app/build/outputs/apk/debug/app-debug.apk 2>&1 | tail -2; \
   adb shell am force-stop com.transitteam.app; sleep 1; \
   adb shell monkey -p com.transitteam.app -c android.intent.category.LAUNCHER 1 >/dev/null 2>&1
   ```
   Confirm the bundle embedded: `unzip -l <apk> | grep index.android.bundle` (≈2 MB entry).
5. Screenshot with `android-shot`.

## DDEV-backed runs (credentialed flows)
The emulator reaches `tq-pro-teams-php8.ddev.site` via the **DNS responder + 10.0.2.2 NAT**, NOT
`adb reverse` (its relay is broken here). Run `node scripts/emulator-ddev-dns.js` and boot the
emulator with `-dns-server 127.0.0.1`; Avast Web Shield must be OFF. See docs/SMOKE_TESTING.md.

## Dead-ends seen in past sessions — avoid
- Blaming the embedded bundle for stale UI → the real cause is Metro unreachable (Avast). Fix Metro.
- `adb exec-out screencap -p > f.png` → stale/0-byte frames. Use `android-shot`.
- Foreground `for i in …; do sleep …` build polling → run the build with run_in_background instead.
